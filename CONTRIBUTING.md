# Contributing

1. Create or select a GitHub issue with acceptance criteria.
2. Create a short-lived branch from `main`.
3. Implement the smallest coherent change.
4. Add unit and integration tests.
5. Run backend and frontend quality checks.
6. Document security and data-governance effects.
7. Open a pull request using the repository template.

Do not commit active malicious URLs, private threat intelligence, secrets, raw provider feeds, personal data, or unapproved serialized models.
